## Gotchas & things to know

1. **`ontology-curator` is NOT renamed.** The research reports sometimes refer to it as `ontology-maintainer`. Keep the directory and YAML name as `ontology-curator`; just add "maintain"/"maintenance" synonyms to the description and to `CONVENTIONS.md § Shared Terminology`.

2. **Description rewrites must preserve existing trigger keywords.** Invariant #4. Run the routing-validation helper after Wave 2 to verify.

3. **Safety hooks are active.** `guard-sed-ontology.sh` will block any bash command matching `(sed|awk|perl).*\.(ttl|owl|rdf|ofn|owx)`. Use `uv run python` or `robot` for anything that touches Turtle.

4. **Ruff strictness is high.** `pyproject.toml` enables E, W, F, I, N, UP, B, A, ASYNC, COM, C4, DTZ, T10, ISC, ICN, PIE, PT, RSE, RET, SLF, SIM, TID, TCH, ARG, PTH, ERA, PL, TRY, FLY, PERF, RUF, S. For markdown-heavy generator scripts (like `build_research_bundle.py`), file-level `# ruff: noqa: E501, RUF001` is pragmatic for string-literal tables. Keep real lint issues fixed.

5. **Pre-commit hooks mutate files.** Trailing-whitespace, end-of-file, and ruff-format will auto-modify staged files and exit non-zero to force you to `git add -u` and re-commit. Plan for a second `git add` after the first `git commit` attempt.

6. **Sevocab CQ tests are the canary for Wave 4 DoD.** Run `uv run python ontologies/skygest-energy-vocab/scripts/run_cq_tests.py` at the end of Wave 4 to confirm the refactor didn't break anything downstream.

7. **`af63239` is history, not main's tip.** The plan text mentions `af63239` as the branching point. Use `main`'s current tip (`$(git rev-parse main)`) in the new session — main has moved since the plan was drafted.

8. **The sky-316 uncommitted work stays sky-316's.** Do NOT merge it into the refactor branch. The research-artifacts commit travels with sky-316 when that PR eventually merges to main.
